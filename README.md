# Telegram-бот «Ежедневный фокус»

MVP-проект объединяет Telegram-бота и простую мини-платформу для курса.

Бот помогает выбрать главную задачу на день, подготовить идею ролика и открыть обучающий Telegram Mini App. Mini App получает модули, уроки и блоки уроков из backend на Flask и SQLite. Администратор может добавлять, редактировать и удалять модули, уроки и блоки уроков через страницу `/admin`.

Для локального запуска и деплоя используется единая команда:

```bash
python start.py
```

`start.py` запускает Flask backend и Telegram-бота одновременно. Backend работает в отдельном потоке, а Telegram-бот остаётся в основном потоке процесса.

## Возможности

- `/start` — приветствует пользователя, скрывает старую нижнюю клавиатуру и включает кнопку меню `Открыть` рядом с полем ввода.
- `/help` — показывает список команд.
- `/plan` — запускает диалог: главная задача и три шага.
- `/focus` — даёт короткий совет по концентрации.
- `/check` — предлагает отметить выполнение задачи кнопками.
- `/content` — спрашивает тему и готовит боль аудитории, обещание, пять хуков и структуру ролика.
- `/app` — включает кнопку меню `Открыть` для запуска обучающего Mini App.
- `/admin` — отправляет ссылку на админ-панель курса, если `WEBAPP_URL` настроен.
- `/cancel` — отменяет текущий незавершённый диалог.

Планы дня хранятся только в памяти процесса. Они исчезают после перезапуска сервиса и автоматически считаются неактуальными на следующий день.

## Стек

- Python 3.10+
- `python-telegram-bot`
- `python-dotenv`
- Flask
- SQLite через стандартный модуль `sqlite3`
- Mini App: HTML, CSS, JavaScript без сборщика
- Уроки состоят из упорядоченных блоков: `text`, `image`, `video`

## Установка локально

Откройте терминал в корне проекта и создайте виртуальное окружение:

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

Для macOS или Linux:

```bash
source .venv/bin/activate
```

Установите зависимости:

```bash
python -m pip install -r requirements.txt
```

## Настройка `.env` локально

Создайте локальный файл `.env` из примера.

В PowerShell:

```powershell
Copy-Item .env.example .env
```

В macOS или Linux:

```bash
cp .env.example .env
```

Заполните значения:

```dotenv
TELEGRAM_BOT_TOKEN=1234567890:your_real_token
WEBAPP_URL=https://your-public-site.example
ADMIN_PASSWORD=123456
DATABASE_PATH=course.db
```

`TELEGRAM_BOT_TOKEN` — токен Telegram-бота от BotFather.

`WEBAPP_URL` — публичный HTTPS-адрес Mini App. Локально можно открыть backend напрямую по `http://127.0.0.1:8000`, но Telegram Mini App требует HTTPS-адрес.

`ADMIN_PASSWORD` — пароль для входа в админ-панель и для admin API.

`DATABASE_PATH` — путь к SQLite-файлу. Локально можно оставить `course.db`.

Файл `.env` исключён из Git. Не публикуйте токены и пароли.

## Локальный запуск всего проекта

Запустите из корня проекта:

```bash
python start.py
```

При первом запуске автоматически создастся SQLite-база и таблицы:

- `modules` — модули курса;
- `lessons` — уроки внутри модулей.
- `lesson_blocks` — текстовые, фото- и видеоблоки внутри уроков.

После запуска откройте:

- Mini App для ученика: [http://127.0.0.1:8000](http://127.0.0.1:8000)
- Админ-панель: [http://127.0.0.1:8000/admin](http://127.0.0.1:8000/admin)

Остановить проект можно сочетанием `Ctrl+C`.

Если нужно запустить только backend без Telegram-бота, для разработки можно использовать:

```bash
python server.py
```

## Как открыть Mini App

Локально откройте [http://127.0.0.1:8000](http://127.0.0.1:8000). Страница загрузит модули через API:

- `GET /api/modules`
- `GET /api/modules/<module_id>/lessons`
- `GET /api/lessons/<lesson_id>`
- `GET /api/lessons/<lesson_id>/blocks`

В Telegram команды `/start` и `/app` включают кнопку меню `Открыть` рядом с полем ввода. Эта кнопка открывает адрес из `WEBAPP_URL`.

## Как открыть админку

Локально откройте [http://127.0.0.1:8000/admin](http://127.0.0.1:8000/admin), введите пароль из `ADMIN_PASSWORD` и нажмите `Войти`.

В Telegram команда `/admin` отправит кнопку со ссылкой на `WEBAPP_URL/admin`, если `WEBAPP_URL` настроен как HTTPS-адрес.

## Как добавить модуль

1. Запустите проект командой `python start.py`.
2. Откройте `/admin`.
3. Введите пароль администратора.
4. В блоке `Создать модуль` введите название.
5. Нажмите `Добавить модуль`.

После этого модуль появится в списке ниже и на странице ученика после обновления.

## Как открыть и закрыть ступени курса

1. Откройте Mini App.
2. На главном экране нажмите на модуль-ступень.
3. Стрелка поменяется с `↓` на `↑`, а под названием появится список уроков.
4. Нажмите на эту же ступень ещё раз, чтобы закрыть список.
5. Можно раскрыть несколько ступеней одновременно.

Если в модуле нет уроков, внутри раскрытой ступени появится текст `Уроков пока нет`.

## Как добавить урок

1. В админке создайте хотя бы один модуль.
2. В блоке `Создать урок` выберите модуль.
3. Введите название урока.
4. При желании введите первый текстовый блок.
5. Нажмите `Добавить урок`.

Урок появится внутри выбранного модуля в Mini App.

## Как собрать урок из блоков

Откройте `/admin`, найдите нужный урок и используйте блок `Блоки урока`.

Чтобы добавить текстовый блок:

1. В форме `Добавить блок` выберите тип `Текст`.
2. Введите текст.
3. Нажмите `Добавить блок`.

Чтобы добавить фото между двумя текстовыми блоками:

1. Добавьте первый текстовый блок.
2. Добавьте блок типа `Изображение` и вставьте URL картинки, который начинается с `http://` или `https://`.
3. Добавьте второй текстовый блок.
4. Если фото оказалось не там, используйте кнопки `↑` и `↓` рядом с блоком.

Чтобы добавить видео:

1. Выберите тип `Видео`.
2. Вставьте URL, который начинается с `http://` или `https://`.
3. Нажмите `Добавить блок`.

YouTube-ссылки показываются внутри урока через встроенный плеер. Остальные видео показываются кнопкой `Открыть видео`.

Чтобы отредактировать блок, измените его тип или содержимое и нажмите `Сохранить блок`. Чтобы удалить блок, нажмите `Удалить блок`.

## Как удалить урок

1. Откройте `/admin`.
2. Найдите нужный урок в блоке `Модули и уроки`.
3. Нажмите `Удалить урок`.
4. Подтвердите удаление в браузере.

После обновления страницы ученика урок исчезнет из списка.

Удаление модуля удаляет и все уроки внутри него.

## Как проверить, что урок появился у ученика

1. В админке создайте модуль, например `Тестовый модуль`.
2. Создайте в нём урок `Тестовый урок`.
3. Откройте адрес Mini App: локально `http://127.0.0.1:8000`, на Railway — ваш публичный домен.
4. Нажмите `Тестовый модуль`.
5. Нажмите `Тестовый урок`.
6. Убедитесь, что отображаются текст, изображения и видео в нужном порядке.

Если в модуле несколько уроков, на странице урока появятся кнопки `Следующий урок` и `Предыдущий урок`. Последний урок модуля ведёт к первому уроку следующего модуля, а первый урок — к последнему уроку предыдущего модуля. Если соседнего урока нет, соответствующая кнопка скрыта.

## Деплой на Railway

Railway должен запускать проект одной командой:

```bash
python start.py
```

### 1. Создайте сервис

1. Загрузите проект в GitHub.
2. В Railway создайте новый проект.
3. Выберите деплой из GitHub-репозитория.

Railway сам установит зависимости из `requirements.txt`.

### 2. Добавьте переменные окружения

В разделе Variables добавьте:

```dotenv
TELEGRAM_BOT_TOKEN=1234567890:your_real_token
WEBAPP_URL=https://your-railway-domain.up.railway.app
ADMIN_PASSWORD=your_strong_admin_password
DATABASE_PATH=/data/course.db
```

`PORT` вручную добавлять не нужно: Railway задаёт его сам. `start.py` и Flask backend читают `PORT` из окружения, а если его нет локально — используют `8000`.

После первого деплоя Railway выдаст публичный домен. Укажите его в `WEBAPP_URL`, например:

```dotenv
WEBAPP_URL=https://telegram-focus-production.up.railway.app
```

### 3. Подключите Volume для SQLite

Чтобы уроки не исчезали после перезапуска Railway:

1. В Railway откройте ваш сервис.
2. Добавьте Volume.
3. Укажите mount path:

```text
/data
```

4. В Variables убедитесь, что стоит:

```dotenv
DATABASE_PATH=/data/course.db
```

Так SQLite-файл будет храниться внутри постоянного Railway Volume.

### 4. Укажите Start Command

В настройках сервиса Railway укажите:

```bash
python start.py
```

После сохранения Railway перезапустит сервис. В логах должны появиться сообщения о запуске Flask backend и Telegram bot polling.

### 5. Проверьте деплой

1. Откройте публичный домен Railway — должна открыться Mini App-страница курса.
2. Откройте `/admin`, например `https://your-railway-domain.up.railway.app/admin`.
3. Введите `ADMIN_PASSWORD`.
4. Создайте модуль, урок и несколько блоков урока.
5. Вернитесь на главную страницу Railway-домена и убедитесь, что урок появился.
6. В Telegram отправьте боту `/start` — рядом с полем ввода должна появиться кнопка `Открыть`.
7. Отправьте `/app` — эта команда должна заново включить кнопку меню `Открыть`.
8. Отправьте `/admin` — бот должен прислать ссылку на админку.

Если Telegram пишет конфликт polling, значит где-то уже запущена другая копия этого же бота. Остановите локальный `python bot.py` или `python start.py`, а затем перезапустите Railway-сервис.

## Ограничения MVP

- Нет оплаты.
- Нет прогресса прохождения уроков.
- Нет авторизации через Telegram.
- Админка защищена простым паролем из переменной окружения.
- SQLite подходит для MVP, но не заменяет полноценную внешнюю базу для большой нагрузки.
- Планы дня бота по-прежнему хранятся только в памяти процесса.
- Для открытия Mini App внутри Telegram требуется публичный HTTPS-адрес.
